## Описание системы
Приложение представляет для себя SaaS платформу для компаний, предоставляя возможность управлять ресурсами компании. Система должна быть спроектирована по multitenant архитектуре. Каждая компания должна иметь собственный tenant. 

# MVP

## 1. Авторизация
- Пользователи не должны создаваться отдельно для каждого tenant. Должна быть одна общая база пользователей, у пользователя должны быть отношения к компаниям один ко многим. 

- При создании нового аккаунта необходимо, чтобы пользователя добавил в систему модератор компании. Модератор - это пользователь системы в компании, который имеет право на редактирование бизнес сущностей внутри его tenant (компании).

- При создании новой компании оставляется заявка на добавление компании в систему. Выбирается бизнес план для компании. Выбирается оплата. В заявку передаются данные компании: ИНН, номер телефона доверенного лица, электронная почта доверенного лица, имя, фамилия, отчество доверенного лица.

- После добавления аккаунта в систему, работник компании может войти в систему с той ролью, которую ему выдал модератор его компании, который добавлял его в систему.

- Также необходимо реализовать возможность восстановления аккаунта через email. При этом в модерацию компании должно отправляться уведомление о том, что пользователь восстановил свою учетную запись и необходимо сделать его дополнительную проверку перед тем, как давать возможность оставлять новые заявки. В то же время для пользователя должна открываться страница с содержимым, объясняющим ему, что его учетная запись находится на проверке модерации и пока он не может ничего сделать.

Авторизацию проще всего будет реализовать с помощью supabase на бекенде


## 2. Ресурсы
В контексте одной компании может проводиться комплекс операций над сущностью ресурса в зависимости от роли пользователя. Модераторы могут: добавлять новые ресурсы для компании, редактировать количество ресурсов, редактировать старые ресурсы, удалять существующие ресурсы, устанавливать определенный уровень пользователя, начиная с которого можно арендовать тот или иной ресурс (например уборщик не может арендовать корпоративный автомобиль). Также модераторы могут добавлять в компанию новых пользователей, вводя его персональные данные. В случае, если персональные данные такого пользователя уже есть в системе (проверка должна проходить по номеру телефоа или почте), то не нужно создавать нового пользователя, нужно добавить к пользователю еще одну компанию. Если пользователя не существует - нужно создать нового и добавить ему в отношения эту компанию.

Каждый пользователь должен иметь возможность видеть историю своих аренд и иметь возможность видеть то, что он арендовал. Должно отображаться в виде календаря.

Аренда должна производиться через заявку в поддержку. Поддержка - специальная роль пользователя, которая должна иметься во всех компаниях и должна иметь возможность подтверждать или отказывать в аренде ресурса. При любом результате должен присылать уведомление на почту пользователя. 

### Для всех пользователей:
Пользователи могут арендовать ресурсы компании. Для этого он должен перейти на страницу со списком ресурсов доступных для аренды. Ресурсы, которые недоступны для пользователя из-за того, что у него слишком низкая роль не должны отображаться для пользователя. Выбрать даты для аренды можно с помощью календаря. Даты, в которые все ресурсы данного типа будут заняты, необходимо заблокировать для выбора. При открытии страницы ресурса, который недоступен для выбранного пользователя, необходимо показать сообщение о том, что данный ресурс закрыт для аренды для этого пользователя.

## Роли пользователей
Роли пользователей настраиваются модераторами системы. Роли должны иметь определенную иерархию. Например: охранник -> работник склада -> начальник склада -> начальник компании. При создании нового ресурса нужно указать, начиная с какой роли пользователь может арендовать ресурс. Также нужно иметь возможность добавлять исключения для некоторых пользователей в ресурсах. Например заблокировать начальнику компании арендовать ведро и т.д.

## Уведомления
Необходимо уведомлять пользователя об аренде в google календаре и по почте, которая привязана к пользователю.

## Статистика
Необходимо иметь страницу со статистикой, которая будет отображаться для модераторов, или для пользователей, которых модераторы сочтут нужными. На основании статистики компания может решить увеличить или уменьшить количество тех или иных ресурсов.
